PRE-REQUIS - COURS 1

KPIs - Business Intelligence - Churn - A/B Testing - Marketing funnel - SQL

KPIs

  • Indicateur clé de performance
  • Evaluer l'efficacité des actions par rapport aux objectifs
  • 2 catégories :
    - Activités du service ou de l'entreprise (Activity Metrics) : nb d'employés, nb de produits, etc...
    - Impact des actions sur le marché (Impact Metrics) : part de marché, vetnes en ligne, etc...
  • Indispensable pour piloter l'activité, analyser son positionnement par rapport à la concurrence, identifier les axes d'amélioration
  • Fonctionnement :
    - Identifier le besoin auxquels ils répondent
    - Les aligner avec la stratégie de l'entreprise et ses objectifs
    - Aider à fournir des plans d'action
  • Attention à la pertinence du nb de KPIs à présenter

Business Intelligence

  • Ensemble des outils et méthodes visant à transmettre les infos pertinentes au manager
  • Base = Collecte des données : qualité, bon format
  • Données stockées dans un serveur informatique centralisé (Data warehouse) puis distribuées à des utilisateurs en fonction de leurs besoins
  • Exploiter les infos dispos : analyser les données + facilement en les restituant intuitivement
  • Data visualisation = représentation des données visuellement et communication (graphiques, camemberts, diagrammes, etc...)
  • Data storytelling = raconter une histoire avec les données
  • 3 étapes :
    - Data discovery : collecter, consolider, modéliser
    - Data visualisation : restituer, représenter
    - Data storytelling : restituer, raconter
  • Attention à la pertinence du nb de KPIs à présenter

Churn

  • Taux de churn = taux d'attrition
  • Indicateur mesurant le volume de clients perdus sur une période donnée
  • KPI mesurant l'efficacité de la stratégie de l'entreprise
  • Autres taux :
    - Taux de fidélisation (rétention) : % de clients fidèles
    - Taux de réachat : nb de clients repassant commande après le 1er authentification
    - CLV (Customer Life Value) : profits générés par un client moyen; + il est élevé, + l'acquisition d'un nouveau client sera rentable sur la durée

A/B Testing

  • Méthode de comparaison statistique des ≠ variantes de pages de sites par rapport à une version de base
  • Identification de la performance selon un objectif défini
  • A/B testing = 2 variantes sur la même URL
  • Split testing = 2 variantes sur une URL ≠ mais pas visible par l'utilisateur → plutôt utilisé pour des gros changements
  • Tests multi-varié (MVT) : plusieurs changements mais nécessite un gros volume de trafic → gain de temps ou impact de chaque variante
  • Tests A/A : même variante
    - Permet d'identifier si taux de conversion similaire et valider le positionnement
    - Valide la performance de A/B testing
    - Définit les valuers de références pour les tests futures et met en place un taux de référence
  • Tests multi-pages : test des modifications sur plusieurs éléments se trouvant sur des pages ≠
  • Utilité du test A/B : vérification d'hypothèses en testant direct sur la population
    - augmenter la conversion
    - fidéliser les visiteurs
    - mieux les connaître
    - prendre des décisions sur des résultats
    - optimiser le temps
  • Test Baudit ou test à allocation dynamique de trafic :
    - modifie la répartition du trafic en fonction des 1ers résultats observés
    - redirige graduellement une proportion croissante de votre trafic vers la variante gagnante
  • Possibilité de cibler les tests avec la segmentation d'audience

  • Client-side : Equipe Marketing pour optimser la conversion
    - modification directe sur le navigateur : tests créés et personnalisation du Front-end
    - code source de la page d'origine est transféré du serveur jusqu'au navigateur
  • Server-side : Equipe technique
    - créer du côté de l'architecture back-end
    permet de contrôler tous les éléments, tester de manière + approfondie
  • Hybride : Mise en place des modifications importantes côté server-side et ajout de KPIs ou modifications simples côté client-side

  • Méthode de calculs statistiques :
    - Méthode fréquentiste : Fiabilité des résultats avec taux de confiance d'obtenir le même résultat en reproduisant l'expérience (seulement à la fin du test)
    - Méthode bayésienne : Probabilité des r&sultats dès le lancement du test. La confiance dan la probabilité qu'une variante soit donnée gagnante augmente à chaque conversion supplémentaire

  • Arrêt du test pour résultat fiable :
    - Atteindre un indicateur fiable (taux de 95%) ou suivre la probabilité de succès
    - Taille d'échantillons suffisante : représentatif de l'audience
    - Période de test bien définie pour lisser les résultats variant selon les facteurs inctrôlables
    - Stabilité des résultats : attendre que le taux de conversion soit linéaire

Marketing funnel

  • Parcours réalisé par un utilisateur avant de convertir un objectif défini (achat, téléchargement, inscription)
  • Modéliser le parcours pour identifier le besoin
  • 3 grandes étapes dans la relation visiteur/site :
    - Prise de conscience : besoin, envie → entrée
    - Considération : solution pour répondre à ses attentes → milieu
    - Décision : achat → sortie du tunnel
  • Tunnel de conversion optimal :
    - Attirer les visiteurs
    - Transformer en prospects
    - Convertir les prospects en clients
    - Fidéliser

SQL

  • Modèle relationnel : représenter et manipuler les données
  • Relation = table / Tuple = ligne d'une table / Attribut = colonne
  • Schéma = Ensemble d'attributs / Domaine = Ensemble de valeurs que peuvent prendre les attributs
  • Clé = unique, permet d'identifier un tuple → groupe d'attributs minimum déterminant le tuple
  • Clé primaire = clé artificielle telle que l'identifiant incrémenté automatiquement
  • Clé ≠ NULL
  • Clé étrangère : colonne faisant référence à une clé primaire d'une autre table
  • Plusieurs cardinalités :
    - 1 à plusieurs (ex : 1 pomme 1 personne)
    - Plusieurs à 1
    - Plusieurs à plusieurs (ex : pls pommes pour 1 personne)
    - 1 à 1 (ex : 1 pomme = 1 personne)
  • Table d'association : Plusieurs à plusieurs. 2 clés étrangères référencant 2 objets de tables ≠
  • Projection : sélection d'attributs en éliminant les autres (tjs après la restriction)
  • Restriction : sélection de tuples selon certaines conditions
  • Union : ensemble des tuples de 2 tables
  • Différence : R1-R2 donne que les tuples restants de R1 et inversement pour R2-R1
  • Intersection : table avec les tuples contenus dans R1 ET dans R2
  • Produit cartésien : toutes les combinaisons possible entre les tuples de R1 et de R2
  • Division : R1 / R2 (ay nt un attrivut commun) est une table avec tous les attributs de R1 qui n'appartenait pas à R2 avec l'ensemble des tuples donnant tjs un tuple R1
  • Jointure : colle 2 tables
    - interne : liaison de 2 tables selon une condition (clé étrangère)
    - externe : si une ligne des tables n'a pas la valeur (= NULL) d'un des conditions → pas dans la table
    - externe à gauche : on garde les lignes de la 1ère table si pas de lien avec la table 2
    - externe à droite : on garde les lignes de la 2e table si pas les infos de la table 1
    - totale : toutes les lignes sont gardées
    - naturelle : si les colonnes à lier des 2 tables ont exactement le même nom
  • Agrégation : groupe d'attributs de partitionnement + une ou plusieurs fonctions d'agrégation (moyenne, count, sum, etc...)
